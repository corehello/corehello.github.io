Gentoo install issues I met

Recently I try to install Gentoo which is automatically optimized and customized for just about any application or need. Details in About gentoo

Firstly, I want to install gentoo via USB driver. But It prompt a error message like "can not found mountpoint balabala....". So I turn to install it via VM.

I install Gentoo on fedora 20 kvm with 1 cpu and 1G RAM, add the minimal CD in kvm as a cdrom and boot from cdrom, then I install it refer to gentoo quick install

Issue 1: install kernel
According to the guide, seems some module will not be installed automatically. so I use genkernel to do all the things. 

  #genkernel --splash --no-install --no-clean --menuconfig all
Details refers to genkernel detail

Since gentoo now use grub2 than grub, so something should be changed to match grub2, generate config file via grub2-mkconfig with grub2 for gentoo

And I also install kde for gentoo, and also met some issues resolved

Install kde for gentoo via kde for gentoo

  • Issue 1: can not find /etc/conf.d/xdm, solution-> install xorg-server
  • Issue 2: when emerge package it prompt message like configration file need update, solution-> run `etc-update`

And finally I install the basic kde environment by myself with google.

Time statics: install kernel take about 5 hours I remember. install kde take about 8 hours I remember.

Conclusion: Gentoo is realy a nice OS for the world, but means it take more time to handle it, it use the best package manager portage and best performance, it give me help to understand how a OS is running and configuration, also I take to many time to solve issue which I met when install it.

Most important: When got a error, do not google immediatly, the error offen come with the detail help to solve this issue with you, so do not worry to read so many words, It is realy helpful to solve the issue.
